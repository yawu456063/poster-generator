# Poster Generator — 海报生成技能

根据用户提供的内容信息，生成一张排版精美、视觉现代的 HTML 海报。内置六套视觉系统，每套三个主题色变体（共 18 种预设），六种布局骨架可选。

## 六套风格系统

### Style A · 极简雅致（默认）

- 大面积留白，纯白/浅灰背景，细线条分隔
- 超细字重标题（weight 200-300）+ 无衬线字体
- 装饰：极细横线、小圆点、小方块
- 适合：艺术展览、设计分享、高端品牌、读书会、文创活动
- 模板：`assets/template-minimal.html` · 主题色：`references/themes.md` A1-A3

### Style B · 商务专业

- 深蓝/深灰主色调，金色/银色点缀
- 结构感强，信息层次清晰，字重对比明显
- 装饰：几何色块、渐变条带
- 适合：企业发布会、行业峰会、商业活动、培训讲座
- 模板：`assets/template-business.html` · 主题色：`references/themes.md` B1-B3

### Style C · 科技未来

- 深色背景（接近黑色），荧光色点缀（蓝/青/紫）
- 发光效果、渐变文字、网格线
- 等宽字体做元数据，Inter 做标题
- 适合：技术大会、黑客松、产品发布、AI/区块链活动
- 模板：`assets/template-tech.html` · 主题色：`references/themes.md` C1-C3

### Style D · 活力缤纷

- 明亮暖色背景，圆润元素、大圆角、多彩色块
- 活泼字体搭配，友好温暖
- 装饰：大色块、波浪、圆形、不规则形状
- 适合：社区活动、工作坊、生日派对、市集、年轻品牌
- 模板：`assets/template-vibrant.html` · 主题色：`references/themes.md` D1-D3

### Style E · 节日庆典

- 深红/暗金主色调，中国风元素
- 衬线标题（Noto Serif SC），有仪式感
- 装饰：金色边框、传统纹样、灯笼感元素
- 适合：年会、新年、开业、周年庆、颁奖典礼
- 模板：`assets/template-festive.html` · 主题色：`references/themes.md` E1-E3

### Style F · 童趣复古字绘

- 米白旧纸背景，复古丝网印刷颗粒，低饱和黄橙/青绿/湖蓝配色
- 主视觉是超大手绘中文字体，像儿童绘本、剪纸、木刻或手工拼贴
- 装饰：小朋友剪影、秋千、气球、纸飞机、滑梯、小印章，装饰必须融入字形或留白
- 适合：儿童节、亲子祝福、幼儿园活动、童书/绘本、生日祝福、童趣艺术海报
- 不适合：严肃、正式、商务、政务、峰会、金融、医疗、讣告、危机公告等场景
- 模板：`assets/template-retro-child.html` · 主题色：`references/themes.md` F1-F3

## 何时使用

**合适的场景**：

- 活动海报（大会、沙龙、工作坊、发布会）
- 邀请函（年会、开业、婚礼）
- 宣传图（课程推广、产品发布、品牌活动）
- 促销海报（限时活动、打折通知）
- 概念发布（新品预告、品牌理念）

**不合适的场景**：

- 需要精确像素级还原设计稿（用 Figma）
- 大量复杂插图/手绘（用专业设计工具）
- 动态视频海报（用视频工具）

## 工作流

### Step 1 · 需求澄清（动手前必做）

**如果用户已经给了完整信息（至少有标题 + 风格/场景线索）**，跳过直接进 Step 2。

**如果用户只给了模糊想法**，用以下问题对齐。一次最多问 3 个最关键的问题，不要逐条追问：

#### 6 问澄清清单

| # | 问题 | 为什么要问 |
|---|------|-----------|
| 1 | **什么风格？**（极简/商务/科技/活力/节日/童趣复古） | **必须先确定**，决定用哪个模板 |
| 2 | **什么场景？**（活动/邀请/宣传/促销/概念） | 决定布局选择和信息密度 |
| 3 | **输出尺寸？**（竖版/正方形/横版，默认竖版） | 决定模板和布局 |
| 4 | **主题色偏好？** | 从对应风格的 3 个预设里选一个 |
| 5 | **必须包含的信息？**（时间/地点/嘉宾/正文/二维码占位） | 决定布局密度 |
| 6 | **有没有硬约束？**（必须包含 XX / 不能出现 YY） | 避免返工 |

#### 风格选择参考

| 如果用户说… | 推荐风格 |
|-------------|---------|
| "简约""干净""高级感""文艺" / 不指定 | **A · 极简雅致** |
| "正式""企业""商务""峰会" | **B · 商务专业** |
| "科技""AI""代码""赛博""黑客松" | **C · 科技未来** |
| "有趣""活泼""年轻""社区""派对" | **D · 活力缤纷** |
| "年会""新年""节日""开业""庆典" | **E · 节日庆典** |
| "儿童节""童趣""亲子""幼儿园""绘本""可爱复古""手绘大字" | **F · 童趣复古字绘** |
| "严肃""正式""庄重""商务""政府""峰会""专业""稳重" | **禁用 F**，优先 **B · 商务专业** 或 **A · 极简雅致** |

**Style F 硬规则**：只在用户明确需要童趣、亲子、儿童节、绘本感、复古手绘大字效果时使用。只要用户提出"严肃画风"、"正式"、"商务"、"庄重"、"专业"等方向，必须取消 Style F，不要用手绘儿童字、气球、滑梯、纸飞机等童趣元素。

#### 信息不足时的策略

- 有标题就能生成，其他字段用合理默认值或省略
- 没有时间/地点 → 省略该区域，不要留占位符
- 没有正文描述 → 省略，让海报更简洁
- 没有风格指定 → 根据内容主题推荐一个

### Step 2 · 选定模板和主题色

#### 2.1 · 拷贝模板

根据 Step 1 确定的风格，使用对应的模板文件：

| 风格 | 模板文件 |
|------|---------|
| A · 极简雅致 | `assets/template-minimal.html` |
| B · 商务专业 | `assets/template-business.html` |
| C · 科技未来 | `assets/template-tech.html` |
| D · 活力缤纷 | `assets/template-vibrant.html` |
| E · 节日庆典 | `assets/template-festive.html` |
| F · 童趣复古字绘 | `assets/template-retro-child.html` |

**注意**：六种风格**不能混用**。每套模板的 CSS 类名、装饰元素、字体系统互相独立。一张海报只选一套。Style F 尤其不能和严肃商务风混用。

#### 2.2 · 选定主题色（18 套预设 · 不允许自定义）

**硬规则**：只允许从 `references/themes.md` 的 18 个预设里选一套，**不接受用户自定义 hex 值**——颜色搭配错了画面瞬间变丑，保护美学比给自由更重要。

**操作**：

1. 基于内容主题推荐一套，或展示当前风格的 3 个变体让用户选
2. 打开 `references/themes.md`，找到对应主题的 `:root` 块
3. **整体替换**模板开头 `:root{}` 里的 CSS 变量
4. 其他 CSS 都走 `var(--...)`，无需任何其他改动

**如果用户坚持要特定颜色**：
- 委婉展示 15 个预设让他选最接近的
- 说明"预设配色经过精心调配，能保证整体视觉和谐"

### Step 3 · 选择布局并填充内容

#### 3.1 · 挑布局

**不要从零写海报结构**。打开 `references/layouts.md`（竖版）或 `references/layouts-horizontal.md`（横版/正方形），里面有现成布局骨架，每种都是完整可粘贴的 HTML 代码块。

**竖版 6 种布局**（详见 `references/layouts.md`）：

| Layout | 用途 | 适合场景 |
|--------|------|---------|
| 1. 中心聚焦 | 内容居中，大量留白 | 简洁公告、品牌活动 |
| 2. 顶部重心 | 标题区大，信息紧凑 | 强调标题/主视觉 |
| 3. 信息密集 | 三段式，层次丰富 | 活动详情多（嘉宾/议程） |
| 4. 左右分栏 | 色块+文字并列 | 需要视觉冲击力 |
| 5. 卡片堆叠 | 卡片组织信息 | 多话题/多场次 |
| 6. 大字报 | 超大标题占 70% | 概念发布、艺术展 |

**横版/正方形**（详见 `references/layouts-horizontal.md`）：

| 类型 | Layout | 用途 |
|------|--------|------|
| 横版 | 左右分割 | 左视觉右信息 |
| 横版 | 上下分割 | 上标题下信息 |
| 正方形 | 中心居中 | 社交媒体分享 |
| 正方形 | 四象限 | 多信息点 |

#### 3.2 · 填充内容

从对应 layout 里复制骨架代码，替换为用户提供的实际内容：

1. 替换标题、副标题、badge 文字
2. 填充时间、地点、正文等信息行
3. 删除用户没有提供的可选区域（嘉宾、CTA 等）
4. 修改 `<title>` 标签为实际海报标题

**绝对不要**：
- 留下 `{{变量}}` 格式的占位符
- 保留没有实际内容的空结构
- 硬编码 hex 颜色值（必须走 `var(--xxx)`）

#### 3.3 · 中文标题字号调整

中文方块字视觉面积大，需要适当控制：

| 标题长度 | 推荐字号 |
|---------|---------|
| ≤ 4 个中文字 | 56-64px |
| 5-8 个中文字 | 44-52px |
| 9-12 个中文字 | 36-44px |
| > 12 个字 | 优先精简标题，不得已用 32-36px |

### Step 4 · 对照检查清单自检

生成完**必须**打开 `references/checklist.md`，逐项对照。

**P0 级别必须全部通过**（否则不能交付）：
- HTML 结构完整可独立打开
- 无占位符残留
- 主标题存在且可读
- 配色来自预设
- 文字与背景对比度充足
- 不依赖外部资源（Google Fonts 除外）
- 所有信息行有实际内容

**P1 级别应该通过**：
- 中文标题不溢出
- 信息层次清晰
- 装饰不遮挡文字
- 底部信息栏存在
- 留白充足

### Step 5 · 保存文件

将生成的 HTML 保存到用户输出目录：

```
C:\Users\11365\cola\outputs\poster-<标题关键词>-<日期>.html
```

### Step 6 · 交付与迭代

告诉用户：

1. **文件路径**（Markdown 链接格式）
2. **打开方式**：用浏览器打开，截图即得海报图片
3. **可调整项**：颜色（换预设）、布局（换骨架）、文字、字号

根据用户反馈迭代——模板 CSS 高度参数化，90% 的调整都是改 inline style 或换主题色变量。

## 核心设计原则

> 违反其中任何一条，海报质感都会垮。

1. **克制优于堆砌** — 宁可留白也不要塞满元素
2. **预设优于自由** — 主题色只能选不能自定义，保护美学
3. **层次靠字号和字重** — 主标题 > 副标题 > 正文 > 元信息，用字号/字重/透明度区分
4. **装饰是配角** — 几何装饰不能抢视觉焦点，更不能遮挡文字
5. **信息完整性** — 用户给了什么就放什么，没给的干净省略，不留空壳
6. **对比度是底线** — 深色背景用浅色文字，浅色背景用深色文字，中间态不允许
7. **语气匹配场景** — 童趣复古只服务轻松祝福和亲子场景；严肃需求要收起可爱元素

## 常见错误（踩坑总结）

| 问题 | 原因 | 解决 |
|------|------|------|
| 标题显示成默认字体 | 模板 CSS 类名用错/缺失 | 先 Read 模板确认类名 |
| 颜色不协调 | 自定义了 hex 值 | 只用预设主题色 |
| 文字看不清 | 浅色文字+浅色背景 | 检查对比度，换主题色 |
| 内容溢出海报区域 | 信息太多 | 换信息密集布局，或精简内容 |
| 装饰遮挡文字 | z-index 或 opacity 不对 | 装饰用 `position:absolute` + 低 opacity |
| 留有 {{变量}} | 忘记替换 | 生成后全文搜索 `{{` |
| 中文标题挤成一坨 | 字号太大 | 按字数分档调整 |
