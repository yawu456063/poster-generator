# 海报质量检查清单

生成海报后，按此清单逐项检查。分 4 个优先级：P0 必须全部通过才能交付，P1-P3 按情况修复。

---

## P0 · 致命（必须全部通过）

不通过任何一项则海报不可交付，需立即修复。

| # | 检查项 | 验证方法 |
|---|--------|----------|
| 0.1 | HTML 结构完整，可独立打开 | 文件以 `<!DOCTYPE html>` 开头，有完整的 `<html><head><body>` 闭合标签 |
| 0.2 | 无 `{{变量}}` 占位符残留 | 全文搜索 `{{`，结果为 0 |
| 0.3 | 主标题存在且可读 | `.main-title` 元素存在，字号 ≥ 36px，内容非空 |
| 0.4 | 配色方案来自预设或用户指定 | 不出现未经定义的随机 hex 值；颜色来自 SKILL.md 配色表或用户明确指定 |
| 0.5 | 文字颜色和背景有足够对比度 | 正文对比度 ≥ 4.5:1（WCAG AA）；标题对比度 ≥ 3:1 |
| 0.6 | 不依赖外部资源（Google Fonts 除外） | 无外部 CSS/JS/图片 CDN 引用（`<link>` 只允许 fonts.googleapis.com） |
| 0.7 | 所有信息行有实际内容 | `.info-value` 内不出现空字符串、"xxx"、"待定" 等占位文字 |

### P0 快速检查脚本思路

```
1. 打开 HTML 文件 → 浏览器无报错
2. Ctrl+F 搜索 "{{" → 无结果
3. 目测标题清晰可读
4. 检查 <link> 标签 → 只有 Google Fonts
5. 检查所有 .info-value → 均有真实文字
```

---

## P1 · 严重（影响观感）

不通过会导致海报看起来不专业或有明显瑕疵。

| # | 检查项 | 验证方法 |
|---|--------|----------|
| 1.1 | 中文标题不超过单行容器宽度 | 在 600px 容器中，标题不折行到第三行（最多两行） |
| 1.2 | 信息层次清晰 | 字号递减：标题(≥36) > 副标题(16-20) > 正文(14-16) > 元信息(11-13) |
| 1.3 | 装饰元素不遮挡文字 | 所有 `.deco-*` 元素的 z-index 低于文字层，或 opacity ≤ 0.15 |
| 1.4 | 底部信息栏存在 | `.footer` 元素存在且包含至少一项内容 |
| 1.5 | 整体留白充足，不拥挤 | `.poster` 的 padding ≥ 36px；元素间 margin/gap ≥ 12px |
| 1.6 | 没有文字被截断 | 无 `overflow: hidden` 导致可见文字被切割 |
| 1.7 | 颜色主题一致 | 不出现与主色方案冲突的颜色（如科技蓝主题中出现暖橙色元素） |

---

## P2 · 一般（值得修复）

不通过不影响基本使用，但修复后品质提升明显。

| # | 检查项 | 验证方法 |
|---|--------|----------|
| 2.1 | 字体降级方案完整 | `font-family` 包含 ≥ 3 个回退字体（网络字体 → 系统中文 → 通用族） |
| 2.2 | badge/tag 使用合适 | 内容 ≤ 4 个汉字或 8 个英文字母 |
| 2.3 | 信息行图标风格统一 | 全部使用 emoji，或全部使用 SVG/CSS 图标，不混搭 |
| 2.4 | 正文字号 ≥ 14px | `.description` 和 `.info-value` 的 font-size ≥ 14px |
| 2.5 | 行高 ≥ 1.6 | 正文（`.description`）line-height ≥ 1.6 |
| 2.6 | border-radius 统一 | 同级元素的圆角值一致（如所有卡片都是 12px） |
| 2.7 | 间距节奏一致 | 同级元素间距相等（不出现 16px→24px→12px 这种随机间距） |

---

## P3 · 建议（锦上添花）

通过后海报品质更高，但不通过也不影响交付。

| # | 检查项 | 说明 |
|---|--------|------|
| 3.1 | 有适度 CSS 动画 | 如 badge 呼吸动画、装饰元素淡入，不刺眼 |
| 3.2 | 装饰元素呼应主题色 | `.deco-*` 使用主色的低透明度变体 |
| 3.3 | footer 有品牌或网址信息 | 包含主办方名称、网址、或品牌标识文字 |
| 3.4 | 使用了合适的 letter-spacing | 标题 `-1px ~ 0`，badge/标签 `1-3px`，正文 `0` |
| 3.5 | 视觉重心在上 1/3 区域 | 最大字号元素位于海报上半部分 |
| 3.6 | 装饰元素使用 CSS（非外部图片） | 几何装饰通过 CSS gradient/border/pseudo-element 实现 |
| 3.7 | 配色有明暗层次 | 同一色系至少有 2 个明度变体（如主色 + 主色半透明） |
| 3.8 | 存在视觉引导线 | divider、渐变线或布局动线引导视线从上到下 |

## 使用方式

### 生成后自检

生成 HTML 文件后，按顺序检查：

1. **P0 全过** → 可以交付
2. **P1 有不通过** → 修复后交付
3. **P2/P3** → 记录但不阻塞交付，下次优化

### 常见 P0 失败原因 & 修复

| 失败项 | 常见原因 | 修复方式 |
|--------|----------|----------|
| 0.2 占位符残留 | 模板复制后忘记替换 | 全局替换所有 `{{xxx}}` |
| 0.4 配色不合规 | 随手写了个颜色值 | 使用 SKILL.md 配色表中的值 |
| 0.5 对比度不足 | 浅色文字 + 浅色背景 | 深色背景用 `rgba(255,255,255,≥0.8)` 的文字 |
| 0.7 空内容 | 用户没提供某字段 | 省略该信息行，或用合理默认值 |

### 对比度速查

| 背景 | 文字最低亮度 | 常用安全值 |
|------|-------------|-----------|
| `#0a0f1e`（深蓝黑） | `rgba(255,255,255,0.8)` | 正文用 0.8+，标题用 1.0 |
| `#ffffff`（纯白） | `#333333` | 正文用 #333-#666，标题用 #1a1a1a |
| `#f5f7fa`（浅灰） | `#333333` | 同纯白处理 |
| `#2c1810`（深棕） | `rgba(255,248,240,0.85)` | 暖色系用 cream 色文字 |
